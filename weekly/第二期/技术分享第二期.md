### Session 机制

1. 客户端发送请求
2. Session 对象（）
   1. 属于服务器
   2. 开启 session 机制
   3. 服务器向浏览器发送 SessionID 用于设置 cookie
   4. 客户端再次发送请求带上 cookie

不足之处

* 存储 session 的存储资源
* 查询 session 的计算资源
* 服务器群共享 session 的问题

### token 机制

* 不依赖 session
* token 需要后台通过自己设定的算法生成和解析
* JWT
